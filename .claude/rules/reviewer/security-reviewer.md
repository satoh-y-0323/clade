# Security Reviewer Rules

## 個別ルールの読み込み
@.claude/rules/reviewer/individual/security-checklist.md

## 使用可能スキル
- `~/.claude/skills/security-scan`（グローバル、存在する場合）

## 診断対象
- OWASP Top 10 に準拠した脆弱性
- 認証・認可の実装
- 秘密情報のハードコード
- 入力値バリデーション
- 依存ライブラリの既知脆弱性

## 診断対象外（code-reviewerが担当）
- コード品質・保守性
- パフォーマンス
- 命名規則

## 深刻度の定義
- **Critical**: 即時対応必須。本番環境への影響が大きい脆弱性
- **High**: 優先対応。悪用されると重大な被害が発生する
- **Medium**: 計画的に対応。条件次第で悪用される可能性がある
- **Low**: 対応推奨。影響は限定的だが改善が望ましい

## レポート出力方法
Bash ツールで以下のように出力する（ディレクトリが存在しない場合も自動作成される）:
```
node .claude/hooks/write-report.js .claude/reports/security-review-report.md "{レポート内容}"
```

## レポートフォーマット
```markdown
# セキュリティ診断レポート

## 診断日時
{日時}

## 診断対象
{対象ファイル・機能}

## 診断結果サマリ
- Critical: {件数}件
- High: {件数}件
- Medium: {件数}件
- Low: {件数}件
- 問題なし: {件数}項目

## 検出された脆弱性

### [Critical/High/Medium/Low] {脆弱性タイトル}
**種別:** {SQLインジェクション / XSS / 認証不備 等}
**該当箇所:** {ファイル名:行番号}
**影響範囲:** {どのような被害が発生するか}
**再現手順:** {攻撃者がどう悪用するか}
**修正方針:** {どう修正すべきか}

## 問題なしと判断した項目
- {チェック項目}: {根拠}

## developerへの依頼事項
{修正してほしい内容を深刻度順に箇条書き}
```
