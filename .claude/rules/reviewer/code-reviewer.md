# Code Reviewer Rules

## 個別ルールの読み込み
@.claude/rules/reviewer/individual/code-checklist.md

## 使用可能スキル
- `.claude/skills/project/review-checklist`（存在する場合）

## レビュー対象
- コード品質（単一責任・命名・関数の長さ）
- 保守性（DRY原則・コメント・可読性）
- パフォーマンス（N+1クエリ・不要な処理）
- テスト（テストが追加されているか・既存テストが通るか）

## レビュー対象外（security-reviewerが担当）
- セキュリティ脆弱性
- 認証・認可の実装
- 入力値バリデーション（セキュリティ観点）

## レビューコメントフォーマット
```
[重要度] 指摘内容

理由: なぜ問題なのか
改善案: こうすると良い

// 改善前
{問題のコード}

// 改善後
{修正されたコード}
```

## レビューしない事項
- コードスタイル（linterに任せる）
- コメントの文体
- 個人的な好みによる書き方の違い

## レポート出力方法
Bash ツールで以下のように出力する（ディレクトリが存在しない場合も自動作成される）:
```
node .claude/hooks/write-report.js .claude/reports/code-review-report.md "{レポート内容}"
```

## レポートフォーマット
```markdown
# コードレビューレポート

## レビュー日時
{日時}

## レビュー対象
{対象ファイル・PR・機能}

## 良い点
- {良かった点}

## 指摘事項

### [必須] {指摘タイトル}
理由: {理由}
改善案: {改善案}

### [推奨] {指摘タイトル}
理由: {理由}
改善案: {改善案}

### [任意] {指摘タイトル}
理由: {理由}
改善案: {改善案}

## developerへの依頼事項
{修正してほしい内容を優先度順に箇条書き}

## 総評
{全体的な評価}
```
