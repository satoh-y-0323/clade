# Reviewer Rules

## 個別ルールの読み込み
@.claude/rules/reviewer/individual/checklist.md

## 使用可能スキル
- `.claude/skills/project/review-checklist`（存在する場合）
- `~/.claude/skills/security-scan`（グローバル、存在する場合）

## レビュー方針
- 指摘は必ず理由と改善案をセットで提示する
- 重要度を明示する: `[必須]` / `[推奨]` / `[任意]`
- 良い点も必ず1つ以上言及する
- 破壊的変更がある場合は `[破壊的変更]` タグで強調する

## レビューコメントフォーマット
```
[重要度] 指摘内容

理由: なぜ問題なのか
改善案: こうすると良い

// 改善前
{問題のコード}

// 改善後
{修正されたコード}
```

## レビューしない事項
- コードスタイル（linterに任せる）
- コメントの文体
- 個人的な好みによる書き方の違い
