# /promote コマンド

プロジェクト固有のスキルまたはルールを、
グローバル（~/.claude/）に展開して全プロジェクトで使えるようにする。

## 実行手順
1. プロジェクト固有ファイルを一覧表示する:
   ```bash
   echo "=== プロジェクト固有スキル ==="
   ls .claude/skills/project/ 2>/dev/null || echo "（なし）"
   echo "=== プロジェクト固有ルール（個別）==="
   find .claude/rules -name "*.md" -path "*/individual/*" 2>/dev/null
   ```

2. 以下の形式で候補を提示する:
   ```
   ## グローバル展開候補

   ### スキル
   1. {ファイル名}: {説明の最初の1行}
   2. ...

   ### ルール
   1. {ファイル名}: {説明の最初の1行}
   2. ...

   展開するものを番号で選んでください。
   ※汎用化が必要なプロジェクト固有の記述は自動的に除去します。
   ```

3. 選択されたファイルを汎用化する:
   - プロジェクト名・固有パス・固有変数名を汎用的な表現に置換する
   - 汎用化した内容をユーザーに確認する

4. グローバルに保存:
   - スキル → `~/.claude/skills/{name}.md`
   - ルール → `~/.claude/rules/{category}/{name}.md`

5. 展開結果を報告する:
   ```
   ## 展開完了
   - {ファイル名} → ~/.claude/skills/{name}.md
   次回セッションから全プロジェクトで使用可能になります。
   ```

## 昇格基準の目安
- 2つ以上の異なるプロジェクトで同じ手順が有効だった
- プロジェクト固有の名称・パスを含まずに記述できる
- 他の開発者にも有効と思われる
