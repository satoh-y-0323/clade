# ワークフロービルダー

`/agent-workflow-builder` は、**プログラミングの知識がなくても独自のエージェントワークフローを構築できる**メタエージェントです。

## どんな人向けか

- 事務・営業・企画など、非エンジニア職の方
- 繰り返し行う業務を Claude Code で自動化したい方
- Clade の標準ワークフロー（開発フロー）ではなく、自分の業務に合ったフローを作りたい方

---

## できること

業務内容をヒアリングし、その業務に特化したエージェント群を自動生成します。

**生成されるもの：**
- 業務専用のスラッシュコマンド（例：`/agent-document-reviewer`）
- 各エージェントの指示書（どんな質問をするか、何を出力するか）
- `CLAUDE.md` への自動登録

生成されたエージェントはすぐに使い始めることができます。

---

## 使い方

```
/agent-workflow-builder
```

実行すると、4つのフェーズで進みます：

### フェーズ1: 業務ヒアリング

どんな業務を自動化したいかを聞かれます。

```
どのような業務・作業を Claude Code で効率化したいですか？

例:
- 議事録の作成
- メールの文面チェック
- 週次レポートの集計
- 契約書のレビュー

→ "議事録の作成と、タスクの洗い出しをしたい"
```

### フェーズ2: ワークフロー設計

業務の内容からステップを提案します。

```
以下のワークフローを提案します：

ステップ1: /agent-minutes-collector
  → 会議の参加者・日時・議題をヒアリング

ステップ2: /agent-minutes-writer
  → 議事録の下書きを作成

ステップ3: /agent-task-extractor
  → 議事録からタスクを抽出・担当者・期日を整理

このワークフローでよろしいですか？
  [yes] このまま進む
  [no]  修正したい点を教えてください
```

### フェーズ3: エージェントファイルの生成

承認後、各エージェントのファイルを自動生成します。

### フェーズ4: CLAUDE.md への登録

生成したエージェントが `CLAUDE.md` に自動登録されます。次回から使えるようになります。

---

## 生成されたエージェントの使い方

生成後は、通常のエージェントと同じように呼び出せます：

```
/agent-minutes-collector
```

エージェントが必要な情報を質問し、最終的なアウトプットを出力します。

---

## カスタマイズ

生成されたエージェントは `.claude/commands/` 内の Markdown ファイルです。テキストを編集するだけで動作をカスタマイズできます。プログラミングの知識は不要です。
