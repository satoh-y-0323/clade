# セッション管理

Clade はセッションをまたいで作業状態を自動保存します。翌日でも前回の続きからスムーズに再開できます。

## セッションコマンド

```
/init-session    # 前回のセッション状態を復元する
/end-session     # セッションを保存して終了する
/status          # 現在のセッション状態を確認する
```

---

## Clade のアップデート

```
/update          # Clade 自体を最新バージョンに更新する
```

GitHub リリースの最新版と現在のインストールを比較し、差分の確認・適用・ロールバックができます。

| サブコマンド | 内容 |
|---|---|
| チェック | 最新バージョンと変更内容を確認する（適用はしない） |
| アップデート適用 | 最新版を適用してコミットする（適用前に自動バックアップ） |
| ロールバック | 直前のバックアップコミットに戻す |

::: tip
アップデートは自動的にバックアップコミットを作成してから適用されるため、問題があればロールバックで元の状態に戻せます。
:::

---

## セッション再開の例

翌日プロジェクトに戻ったとき：

```
> /init-session

## セッション再開 (2026-04-05)

### 前回セッション (2026-04-04)
残タスク:
- [ ] コードレビュー指摘対応: 認証ロジックをサービス層に分離する
- [ ] リファクタ後のテスト再実行

前回うまくいったこと:
- 認証フローの実装完了
- JWT トークンの発行・検証ロジックを実装

続きから作業しますか？それとも新しいタスクを開始しますか？
```

前回の残タスク・成功アプローチ・失敗パターンが自動的に表示されます。

## 自動保存されるもの

### セッションファイル（`.claude/memory/sessions/`）

セッション終了時（`/end-session` 実行時）に以下が保存されます：

- **うまくいったアプローチ**（証拠付き）
- **試みたが失敗したアプローチ**
- **まだ試していないアプローチ**
- **残タスク**

### コンテキスト圧縮への対応

長い会話でコンテキストウィンドウが圧縮される際、Clade は自動的にチェックポイントをセッションファイルに記録します。重要な文脈が失われても、セッションファイルから状況を復元できます。

## 学習機能

Clade はセッションを重ねるごとにプロジェクト固有のパターンを学習します。

### スキル化（`/cluster-promote`）

bash-log に蓄積されたコマンド実行履歴を分析し、繰り返し使われているパターンをスキルとして昇格できます。

### ルール化・グローバル昇格（`/promote`）

プロジェクト内で学習したスキルやルールを、複数プロジェクトで有用と判断した場合にグローバルスコープへ昇格できます。**昇格するかどうかは常にあなたが決めます。**

---

## 自動実行されるフック

| イベント | 処理内容 |
|---|---|
| セッション開始 | 前回セッション・メモリ・インスティンクト・スキル一覧をコンテキストに注入 |
| ツール実行前 | 危険コマンドのガード（`rm -rf /` 等をブロック） |
| ツール実行後 | Bash コマンド結果を `bash-log.jsonl` に記録 |
| セッション終了 | セッションファイルの雛形を作成 |
| コンテキスト圧縮前 | セッションファイルにチェックポイントを記録 |
