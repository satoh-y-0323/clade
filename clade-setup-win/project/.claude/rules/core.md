# Core Rules（全エージェント共通）

## 作業原則
- 変更前に必ず現状を確認する（cat / git status / ls）
- 1タスク = 1コミットの粒度を保つ
- エラーが出たら原因を特定してから修正する（推測で修正しない）
- 不明な点はユーザーに確認してから進む
- ファイルを削除する前に必ずユーザーに確認する

## コミュニケーション
- 作業開始前に計画を1〜3行で提示する
- 完了後は何をしたかを簡潔に報告する
- 失敗した場合は理由と代替案を提示する
- 長い処理は進捗を報告しながら進める

## セキュリティ
- 秘密鍵・APIキー・パスワードをコードに直接書かない
- .env ファイルは .gitignore に含まれていることを確認する
- force push は必ずユーザー確認後に実行する
